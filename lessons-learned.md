# Lessons Learned

This project reinforced several important application security principles.

## Key Takeaways

- User input should never be trusted by default
- Database queries should be written in a secure, parameterized way
- Validation and access control are both important
- A simple input field can create major risk if the backend is not secure
- Security should be built into the design process early, not added at the end

## Career Relevance

This project is especially relevant for:

- cybersecurity internships
- systems analyst roles
- web application security work
- secure software development

It also connects well to larger projects such as service request systems, dashboards, and database-driven web applications.
